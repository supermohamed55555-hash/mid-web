How the Web Works

Understanding the architecture, protocols, and communication of the modern internet.

What is the Web?

The Web (World Wide Web) is a system that allows users to access and view websites through a web browser. Websites are made of files like HTML, CSS, and JavaScript that are displayed in a structured way.
الويب (World Wide Web) هو نظام بيسمح للمستخدمين يفتحوا ويشوفوا المواقع باستخدام المتصفح. المواقع دي بتكون عبارة عن ملفات زي HTML و CSS و JavaScript وبتتجمع عشان تظهرلك صفحة كاملة.

Clients vs Servers

Client: Your device and browser.
Server: A powerful computer storing the website.
الـ Client: هو جهازك (موبايل أو لابتوب) + المتصفح اللي بتستخدمه.
الـ Server: هو جهاز قوي بيخزن المواقع ويبعتها لما تطلبها.

How it Works (Step-by-Step)

1. Check DNS for Server IP.
2. Send HTTP Request.
3. Server processes it.
4. Server sends website files.
5. Browser displays the page.
1. المتصفح يسأل DNS عن عنوان السيرفر.
2. يبعت طلب (Request) للسيرفر.
3. السيرفر يستقبل الطلب ويعالجه.
4. السيرفر يبعت ملفات الموقع.
5. المتصفح يجمع الملفات ويعرض الصفحة.

Main Components

Internet Connection: The network allowing communication.
DNS: Translates names (google.com) to IPs.
TCP/IP: Rules for data transmission.
HTTP: The language between client and server.
اتصال الإنترنت: ده اللي بيخلي جهازك يقدر يبعت ويستقبل بيانات.
الـ DNS: زي دليل التليفونات، بيحول اسم الموقع لعنوان رقمي (IP).
الـ TCP/IP: قواعد أو قوانين بتنظم إزاي البيانات تتحرك بين الأجهزة.
الـ HTTP: هو اللغة اللي بيتكلم بيها المتصفح مع السيرفر.

Packets

Header
Payload

Data is split into small chunks for efficiency.

Data is divided into small pieces called packets. Each has a Header (info) and Payload (data).
البيانات على الإنترنت مبتتبعتش مرة واحدة، بتتقسم لأجزاء صغيرة اسمها Packets. كل جزء فيه معلومات (Header) والبيانات نفسها (Payload).

Request & Response

GET / HTTP/2
HTTP/2 200 OK
Request: Browser asks for data.
Response: Server sends requested data.
الطلب: المتصفح بيبعت طلب عشان يجيب صفحة.
الرد: السيرفر بيرد بالبيانات المطلوبة.
